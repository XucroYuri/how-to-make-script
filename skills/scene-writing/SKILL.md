# Scene Writing

Use this skill when outline-level intent needs to become scene-level dramatic action.

## Workflow
1. Define scene function and conflict.
2. If this is an opening scene, decide its primary opening job before drafting.
3. Build the change point.
4. Carry dialogue through playable action or inline performance so no beat floats as bare dialogue.
5. Deliver a scene card, scene draft, or screenplay draft.
6. If voice or liveness constraints are explicit, load the smallest expression lens before drafting lines.

## Output Contract
- `scene_card`: a compact scene design with function, conflict, and change.
- `scene_draft`: a scene-level screenplay draft that advances the story in visible, playable steps.
- `screenplay_draft`: a multi-scene screenplay draft with handoff-safe scene blocks.
- Every scene should change at least one of these: information, relationship, power, or action conditions.
- Every dialogue beat needs a playable carrier: inline performance or an adjacent action beat.
- `VFX` / `SFX` / `BGM` are optional, but when used they must stay in fixed labels rather than loose notes.
- If voice calibration is unresolved, prefer a short guide first over generic neutral prose.

## Posture-Adaptive Guidance

**When `certainty = lost`:**
Ask: "Who is in this scene and what does each of them want that the other person is blocking?" Two characters, two blocked wants. That is enough to start. Do not ask for scene function, beat type, or outline position.

**When `source = discover`:**
Start from the image or feeling that the scene carries. "What does this scene feel like in its last moment, before anyone speaks?" Let the ending image pull the scene into existence.

**When `source = construct`:**
Apply the full workflow: define scene function → set conflict → build change point → carry dialogue through playable action → deliver to output contract. No beat floats without a carrier.

**When `source = generate`:**
Set a collision condition and write without planning. "Two people who cannot afford to trust each other need something only the other person can give. Write the first exchange." Do not pre-decide the outcome.

**When `focus = character`:**
Scene function is secondary when character focus is active. Ask: "What does this scene reveal about this character that no other scene could?" Let that revelation design the conflict.

**When `focus = language`:**
Voice separation is the primary concern. Before drafting, derive a two-line register sketch for each speaking character: what they reach for when under pressure, and what they suppress.

**When `focus = audience`:**
Audience experience is the primary concern. Ask: "At the end of this scene, what does the audience know that at least one character does not?" That gap is the scene's audience contract.
