# Character World

Use this skill to make roles and world constraints dramatically useful.

## Workflow
1. Build the character from desire, fear, value order, and contradiction.
2. Pair major capabilities with limits, costs, or misuse risks.
3. Describe triggered behavior patterns, not just static traits.
4. Turn worldbuilding into pressure rules.
5. Clarify rival forces and key relationships.

## Output Contract
- A character-pressure summary that shows what the protagonist wants, fears, and misreads.
- A world-pressure summary that explains what rules matter and how they create conflict.
- A relationship map that identifies allies, rivals, and pressure points.

## Posture-Adaptive Guidance

**When `certainty = lost`:**
Ask one question: "What does this character want so badly that it embarrasses them to admit it?" Skip the full character framework. One honest answer unlocks more than a completed worksheet.

**When `source = discover`:**
Do not start with a character template. Start with a contradiction: what does this person believe about themselves that the story will prove wrong? Let desire and delusion surface before designing the arc.

**When `source = construct`:**
Apply the full character-build sequence: desire → fear → value order → contradiction → triggered behaviour → capability/limit pairing. World rules become pressure constraints, not decoration.

**When `source = generate`:**
Propose a collision: put the protagonist in the situation where their core belief is most directly threatened. Do not plan the outcome. Write what they do first.

**When `focus = character`:**
Primary activation context. Load `ka.character-drive-quartet`, `ka.triggered-behavior-profile`, and `ka.character-arc` first.

**When `focus = world`:**
Lead with world-rule design: what rules create unavoidable pressure for this character type? Treat world logic as a character generator, not as a setting backdrop.
