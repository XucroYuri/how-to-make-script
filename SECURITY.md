# Security Policy

## Supported Versions

Only the latest `main` branch is supported at this stage.

## Reporting a Vulnerability

Do not disclose security issues in public issues or pull requests.

Instead:
1. Prepare a minimal reproduction or affected file list.
2. Describe impact, prerequisites, and any known exploit path.
3. Send the report privately to the maintainers using the repository security contact once the GitHub project is published.

Until a private contact is configured, treat this repository as pre-release and avoid publishing secrets, proprietary scripts, or private client materials in issues.
