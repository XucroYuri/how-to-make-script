# ID Policy

## Prefixes
- `ka.` for `knowledge_atom`
- `wp.` for `workflow_protocol`
- `rb.` for `evaluation_rubric`
- `fx.` for `example_fixture`
- `skill.` for skill manifests

## Rules
- IDs are lowercase ASCII.
- Use dot-separated namespaces.
- IDs are immutable once published.
- Replacements must use a new ID and mention the superseded ID in body text or changelog.
- Filenames may change when needed; IDs must not.
