# Outline Rubric

用于判断大纲是否真的能支撑成片。

大纲阶段最需要防止的是“看起来已经很完整”。因为这时候最容易被事件数量、结构模板和桥段热闹感骗过去。这个 rubric 的作用，就是把判断拉回因果、升级、闭合这几个根骨层面。
如果项目一开始就用了不合适的结构视角，这套表面完整感会来得更快，也更危险。
