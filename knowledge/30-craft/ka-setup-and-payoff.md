# 铺设与回报追踪

铺设和回报是叙事信任的基本单元——你在早期场景中让读者/观众关注某个东西，承诺这个东西有意义，然后在后期场景中兑现这个承诺。对LLM来说，这不是一个可以靠隐式记忆完成的任务。需要显式的跨场景追踪。

## 核心原语：铺设账本

铺设账本是追踪已铺设但尚未回报的叙事承诺的最小结构：

| 字段 | 说明 |
| --- | --- |
| 铺设ID | 可供后续场景引用的简短标识符 |
| 铺设场景 | 铺设所在场景的编号或标题 |
| 铺设内容 | 被叙事关注的物件、信息、情感承诺 |
| 铺设方式 | 如何让读者/观众注意到它（特写、重复、情感重音、对话强调） |
| 计划回报类型 | direct / variant / implicit |
| 计划回报场景 | 暂定回报位置，可为空 |
| 中间强化 | 是否需要在某个中点场景中让这个铺设保持活跃 |
| 状态 | planted / reinforced / paid_off / abandoned |

状态转换：`planted → reinforced → paid_off`，或者 `planted → abandoned`（显式决定放弃）。

## 三类回报

**直接回报**：铺设内容被精确使用。角色在第一幕捡起一块尖锐的石头，第三幕用它划断绳索。物件、信息、技能类铺设的标准回报方式。

**变体回报**：铺设以意外方式回归。角色一直在找一把钥匙，但从未找到——然而在回报场景中，她意识到门从没锁过。铺设被「违反」但戏剧功能被更高阶地满足。

**隐式回报**：情感或主题层面的闭合。第一幕中母亲说「你永远不会回来」，最终场景中女儿回到母亲墓前——没有物件传递，但情感线的铺设得到了回报。

## 空铺设检测

如果一个已登记的铺设被删除，而所有回报场景依然成立，那么：
- 这个铺设是多余的装饰，不是结构性铺设。
- 删除它，保留回报内容（回报本身可能有独立的戏剧功能）。

这个检测防止作者把每个道具都当作需要追踪的铺设，导致账本膨胀。只有那些被叙事关注过的元素——被镜头停顿过、被重复提及过、被赋予了情感重量——才是真正的铺设。
