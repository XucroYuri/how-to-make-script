# Project Surface Map Reference Pack

This pack teaches how a stronger project-surface design differs from a weaker one.

## Case 1: Long-Horizon Feature Development

### Problem

A feature project will be developed over months with repeated premise, outline, rewrite, and review loops.

### Stronger Surface Map

- canonical source:
  - premise slate
  - outline truth
  - character anchors
  - continuity anchors
- runtime state:
  - current rewrite state
  - lane summaries
  - packet trace
- packet layer:
  - route anchor
  - selected truth sources
  - current rewrite focus
  - latest handoff
- review surfaces:
  - rewrite diagnosis
  - continuity review
  - production-reality review
- export surfaces:
  - development packet
  - rewrite packet

### Weaker Surface Map

- one giant “current project file”
- one ad hoc prompt history
- no difference between editable truth and runtime summary
- no inspectable packet
- review comments mixed directly into source text with no gate logic

### Why The Stronger Map Wins

- Humans know what to edit.
- Agents know what to compile.
- Review results are traceable.
- Runtime drift is easier to catch.

### Why The Weaker Map Fails

- Every collaborator edits a different pseudo-truth.
- Old summaries silently override current source.
- Review and generation are impossible to reproduce.

## Case 2: Short-Drama Production Line

### Problem

A short-drama team needs concept, role system, episode directory, episode drafting, self-check, compliance, and export to move fast without losing hook discipline.

### Stronger Surface Map

- canonical source:
  - creative spine
  - role system
  - episode directory truth
- runtime state:
  - current episode state
  - hook distribution review
  - batch writing telemetry
- packet layer:
  - episode-specific brief
  - locked hook type
  - local continuity notes
- review surfaces:
  - per-episode self-check
  - full-season check
  - compliance surface
- export surfaces:
  - domestic format
  - overseas format

### Weaker Surface Map

- one master script file
- a separate chat log full of hidden decisions
- no explicit export mode
- compliance only checked at the very end

### Why The Stronger Map Wins

- Batch progression becomes possible.
- Format switching is explicit.
- Review and export are not hidden afterthoughts.

### Why The Weaker Map Fails

- Episode-level drift accumulates silently.
- Export and compliance become expensive late surprises.

## Non-Dogma Note

Not every screenplay project needs this much surface structure.

The point is not to maximize artifacts.
The point is to make truth, state, packet, review, and export explicit when long-horizon collaboration makes them necessary.
