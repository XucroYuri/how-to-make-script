# Epistemic Stance

This repository should not behave like a screenplay catechism.

Its job is not to prove that there is one correct way to write. Its job is to make creative decisions more legible, more challengeable, and more reusable without pretending that screenplay practice has a single universal truth.

## Core Position

The repository treats screenplay guidance as:

- heuristic, not absolute;
- bounded, not context-free;
- historically situated, not timeless;
- medium-shaped, not medium-neutral;
- challengeable, not authority-protected.

That means an atom, protocol, or rubric can be strong and still be contestable. A useful rule is not the same thing as a universal law.

## What This Changes

Agents and maintainers should:

- prefer multiple viable paths when the constraints do not force one path yet;
- distinguish hard boundaries from soft constraints;
- preserve useful counterexamples even when the default rule survives;
- prefer scope correction when a challenged claim still has a surviving core;
- ask whether a rule is local to a medium, audience, platform, era, or production context;
- avoid collapsing ambiguity too early in discovery and design work.

## False Universal Warning Signs

Treat these phrases as warning lights:

- `always`
- `never`
- `real screenwriting requires`
- `the audience will always`
- `this is the only workable structure`
- `this genre must`

Sometimes those claims are directionally useful. They still need scope, boundary, and counterexample pressure.

## Preferred Repository Behavior

When uncertainty remains meaningful:

- return 2-4 candidate paths instead of one forced answer;
- name the tradeoff that separates them;
- say which constraints would eliminate each path;
- mark which rule is strong, weak, local, or historically contingent.

When a rule is challenged but not entirely broken:

- preserve the smallest surviving core;
- name the failure context explicitly;
- narrow the claim instead of replacing it with an opposite universal.

When constraints are already tight:

- converge hard;
- make the chosen route explicit;
- say what was ruled out and why.

## What This Does Not Mean

Pluralism does not mean:

- every idea is equally strong;
- no guidance can be given;
- all objections are equally valid;
- the repository should become vague on purpose.

The point is not to remove judgment. The point is to make judgment scoped, evidence-seeking, and revisable.
