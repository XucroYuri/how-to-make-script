# 多语种视觉语言层

这个仓库新增这层，不是为了显得“国际化”，而是因为现在很多剧本工作已经不只停在文本内部，而是会继续流向：
- 跨语言团队沟通；
- 分镜 / previz；
- 视频生成；
- 国际协作或多语言视觉 brief。

一旦到了这一步，如果 Agent 只会用模糊形容词讲镜头、灯光、动作和气质，剧本层的判断就会在下游失真。

## 这层解决什么

它解决的是“怎么把视觉意图讲对”的问题，不是“怎么堆更多外语词”的问题。

高价值的多语种视觉语言，不是整本术语词典，而是当前任务真正需要的那一小组词：
- 哪个景别最该保留；
- 运镜到底该怎么说才不歪；
- 哪个文化美学词会真正改变画面质地；
- 哪些词适合用本地语言保留，哪些词反而应该用英文技术锚点更稳。

## 为什么不能默认加载

因为这层一旦整包默认加载，很容易退化成两种坏东西：
- 外语词炫技；
- 把风格、镜头、文化、工具限制混成一大坨。

所以它只该在这些情况下介入：
- 用户明确要多语种镜头语言；
- 任务涉及跨语言视觉 handoff；
- 下游生成或 previz 会因为语言表达失真而掉质量；
- 单靠中文 / 英文当前工作语，已经讲不清真正的气质与执行差异。

## 当前重点语种

- 中文：镜头、运镜、光线、动作密度表达压缩度高。
- 日语：擅长处理留白、停顿、时间感和微妙气氛。
- 韩语：适合高情绪、高关系压强、MV / 韩剧情绪节奏。
- 俄语：适合诗性、纪实、蒙太奇、沉郁与时间质感。
- 西语：适合热感、社会性、魔幻现实和情绪外放场景。
