# Exploration vs Review

Creative work splits into two phases that must stay separate:

1. **Exploration** — widen the option space, test bold moves, suspend soft conformity checks.
2. **Review** — gate, filter, validate what survives against real constraints.

This is a sequencing rule, not a blanket permission slip. Exploration does not suspend hard safety boundaries. Review should not shrink exploration before it starts.
