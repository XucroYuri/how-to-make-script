# Source Map：剧本方法族公开参考

这份文件记录的是“帮助未来协作者定位方法族”的公网入口，不是为了宣布哪套方法是唯一真理。

## 1. Logline、beat 与结构表面层

参考来源：

- [Final Draft: Logline, Story Beats and Brainstorming](https://www.finaldraft.com/learn/loglines-story-beats-brainstorming/)
- [Final Draft: What Is a Story Beat?](https://www.finaldraft.com/blog/what-is-a-story-beat)
- [Final Draft: Building Screenplay Structure](https://www.finaldraft.com/learn/building-screenplay-structure/)
- [Save the Cat essential vocabulary PDF](https://savethecat.com/wp-content/uploads/2019/07/STC_Crash_Course_Essential_Vocabulary.pdf)
- [Save the Cat at-a-glance movie sheet PDF](https://savethecat.com/wp-content/uploads/2013/11/STC-At-The-Movies-At-A-Glance.pdf)
- [StudioBinder: Save the Cat Beat Sheet Explained](https://www.studiobinder.com/blog/save-the-cat-beat-sheet/)
- [StudioBinder: What is a Story Beat in a Screenplay?](https://www.studiobinder.com/blog/story-beat-in-screenplay/)

观察到的信号：

- logline 和 beat 都首先是开发判断工具，而不只是格式工具；
- beat 很重要，但不存在一种 beat 载体能诚实覆盖所有项目；
- 真正有用的不是“有没有 beat”，而是“有没有用对承载变化的结构容器”。

仓库吸收方式：

- 把“结构族选择”和“beat 载体选择”分开；
- 把“伪 logline 预警”单独抽出来，防止漂亮句子冒充故事发动机。

## 2. 开场设计与注意力入口

参考来源：

- [StudioBinder: Screenplay Opening Scene Examples](https://www.studiobinder.com/blog/screenplay-opening-scene-example/)
- [StudioBinder: In Medias Res](https://www.studiobinder.com/blog/in-medias-res-definition-examples/)

观察到的信号：

- 好开场承担的工作并不相同：有的负责冲突，有的负责问题缺口，有的负责角色承诺或 premise promise；
- “有钩子”和“开场工作正确”不是一回事。

仓库吸收方式：

- 不把开场写成招式菜单，而是写成“主工作选择”；
- 开场必须兑现观看承诺，而不是只争夺一秒注意力。

## 3. 神话型与非神话型结构族

参考来源：

- [Joseph Campbell Foundation: The Collected Works / The Hero with a Thousand Faces](https://www.jcf.org/learn/joseph-campbell-collected-works)
- [Joseph Campbell Foundation: Myth, Campbell & Film](https://www.jcf.org/post/myth-campbell-film)

观察到的信号：

- Hero's Journey 对 quest / transformation 型项目仍有强影响力；
- 但它只能作为一种结构族，而不能吞掉其他推进机制。

仓库吸收方式：

- 神话型结构只作为候选结构族之一进入仓库；
- 结构选择仍以故事推进引擎为先，而不是以声望或教材熟悉度为先。

## 4. 剧本到视觉执行的边界

参考来源：

- [StudioBinder: Screenplay Formatting / Path to Production](https://www.studiobinder.com/screenplay-formatting/)
- [StudioBinder: Storyboard template PDF](https://www.studiobinder.com/wp-content/uploads/2016/06/StudioBinder-Storyboard-Template-2-Frames-Landscape.pdf)
- [StudioBinder Support: Scene heading best practices](https://support.studiobinder.com/en/articles/3032128-best-practices-when-formatting-a-scene-heading-in-a-screenplay)

观察到的信号：

- screenplay、shot planning、storyboard、production packet 是相邻层，不是同一层；
- 如果上游剧本结构还可追溯，下游 pre-production 和 bridge 层会稳很多。

仓库吸收方式：

- 在 bridge 层强调 source traceability、visible asset grounding 和容器边界诚实；
- 不把 storyboard 或 clip 层细节误写成 screenplay 层的单一权威来源。

## 采用规则

仓库可以吸收：

- 可长期复用的抽象；
- 协作者可访问的公开参考入口；
- 有助于未来扩展的 source map。

仓库不应吸收：

- 私有本地路径；
- 私有工作笔记的隐藏权威地位；
- 只适用于某个本地流程的 prompt 壳或模板语法。
