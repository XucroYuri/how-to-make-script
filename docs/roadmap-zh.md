# Roadmap

## Phase 1

- 初始化仓库元数据和社区协作文件。
- 冻结 taxonomy、routing、ID policy 和 schema 契约。
- 增加根 skill 和第一批薄子 skill。
- 建立 discussion-first 的社区输入方式，让反驳和 field note 不会过早被硬塞进 issue。

## Phase 2

- 在故事、场景、对白、节奏和 rewrite craft 上扩展基础 knowledge atom。
- 深度覆盖前三个重点簇：
  - narrative screenwriting
  - commercial / branded scripting
  - interactive / branching narrative
- 增加受众需求、委托适配、作者成长、平台逻辑和历史演变的现实锚定包。
- 强化双轨编辑系统，让 English 保持 agent-facing，而 Chinese 保持 practitioner-facing。
- 在新增或拆分 knowledge asset 之前，把 Socratic question backlog 作为默认入口关卡。

## Phase 3

- 增加更多 genre pack 和 medium-specific constraint。
- 扩展 case study 和 golden fixture。
- 提高 audience-fit、development strategy 与 writer training route 的 fixture 深度。
- 增加 multilingual visual-language pack 和 screenplay-to-video bridge route，同时避免把仓库变成供应商定制的 prompt 目录。
- 扩展 team-mode blueprint、handoff contract 和 human-gate pattern，覆盖主要剧本协作模型。
- 增加更强的 lint 规则，用来检查互相矛盾或定义不足的 asset link。
- 把贡献者梯度从 issue reporter 扩展到 discussion moderator、triage steward 和 field-note curator。

## Phase 4

- 引入可选的生成式索引和 registry 发布。
- 增加翻译层和多语言知识视图。
- 为下游 agent 系统增加更高级的 rubric-driven rewrite loop。
