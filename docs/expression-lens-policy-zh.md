# 表达校准包策略

仓库现在把“语言风格 / 角色声音 / 活人感”正式视为一层独立能力，但它不是默认对所有请求都开启的总开关。

这点很关键。如果每次写 scene、dialogue、commercial 都把风格资料整包塞进去，系统很快又会回到上下文膨胀和模板化表达。如果完全不把风格明确建模，模型则很容易写出”结构正确但没活人感”的文本。

所以更合理的做法是双层：
- 用户明确要调风格时，走专项输出物 `voice_style_guide`（表达风格指南）；
- 用户仍然要 scene / dialogue / commercial / adaptation 时，只在真的需要时，给主路由补一个轻量的表达校准包。

这不等于所有跨语言或视觉表达问题都应该扔进表达校准包。如果用户真正缺的是多语种镜头语言或下游视觉衔接，就应该进入 `visual_language_pack` 或 `screen_to_video_brief` 这类更贴边界的输出物。

## 这层到底解决什么

它主要解决这些常见失败：
- 人物明明有设定，写出来还是像作者在代言；
- 续写 IP 时，只会模仿口头禅，不会保角色底层逻辑；
- 品牌片语言想高级，结果写成假高级；
- 互动角色承担信息功能太重，台词像 NPC 菜单而不像活人回避。

## 一份能用的表达校准包，至少要同时保住四件事

- `voice anchor`：这个人 / 这个 IP / 这个品牌从什么立场说话，惯常怎么护住自己；
- `lived pressure`：身体压力、关系风险、时间成本怎样进入语言；
- `register envelope`：允许什么语域，禁止什么假感或错位感；
- `variability budget`：哪些变化是活的，哪些变化会直接漂移成另一个人。

这四件事里少一件，结果通常就会变成两种坏东西之一：空泛风格词，或者僵硬模板。

## 设计规则

用户明确要“风格校准 / 声音校准 / IP 一致性 / 活人感处理”时，优先产出 `voice_style_guide`。

如果用户其实还是要成稿，就维持原路由不变，只在以下情况下额外加载表达校准包：
- constraint 里明确出现 `voice_target`、`language_register`、`ip_continuity`、`experiential_depth`；
- 现有文本已经暴露出作者腔、同质腔、假高级、NPC 腔、off-IP；
- 改编或续写工作如果不先锁 continuity anchor，后面根本没法安全写词。

## 要特别避免的失败

- 只有“更自然一点 / 更高级一点 / 更像活人一点”这种空评价；
- 把声音理解成口头禅和标签；
- 用抽象情绪总结替代具身化压力；
- 把品牌调性偷换成千篇一律的奢侈品文案腔；
- 把 continuity 理解成模仿原句，而不是守住角色的判断逻辑；
- 为了风格而无限扩容参考，最后把文本写钝。

## 一句话

表达校准包要改变的是“怎么写”，而不是只增加一层“怎么评价写法”的形容词。
