# 场景图谱

这个图谱不是“剧本类型大全”，也不是“唯一正确写法清单”。

它的作用只有一个：把一个剧本创作任务先按多个维度拆解，再决定应该往哪条路走。也就是说，这里提供的是路由参考、比较参考和问题拆分方式，不是 canon。

机器可读注册表在 [`references/scenario-taxonomy.json`](../references/scenario-taxonomy.json)。

## 怎么用

1. 先看容器：电影、剧集、短剧、广告、互动，还是混合项目。
2. 再看交付物：是要 `premise`、`outline`、`scene_draft`，还是 `rewrite_report`、`boundary_map`。
3. 然后再看受众要什么、故事靠什么跑、行业条件是什么。
4. 如果一个任务本身还没收敛，不要急着限定为单一路线。
5. 如果是混合项目，就保留多个场景卡，不要强行归为一个标签。

## 分类维度

- `medium`：作品容器是什么。
- `format_pressure`：现在要交什么形态的东西。
- `audience_need_state`：观众此刻真正想要什么。
- `narrative_engine`：故事靠什么推动。
- `emotional_contract`：这个作品承诺给观众什么情绪。
- `production_context`：现在是在提案、开发、重写，还是改编。
- `participation_mode`：观众是纯观看、选分支，还是带状态的互动。
- `business_goal`：项目最终想达成什么。
- `genre_pressure`：当前最强的类型承诺是什么。
- `source_relation`：和原作、IP、改编源是什么关系。
- `collaboration_mode`：是单写、编剧室、客户回改，还是委员会审。
- `risk_profile`：这个任务是低风险、中风险还是高敏感。

这些维度是”看问题的角度”，不是”规定答案的法律”。

## 代表性场景卡

### 1. 长片原创开发
- 适用：从一个原创点子往长片 premise / outline 推进。
- 参考片段：`他把火场地图塞进外套里，明知城市已经把那次坍塌称作“可控事故”，还是转身往里走。`
- 为什么更好：有具体物件、有外部压力、有可见风险。
- 为什么弱写法不行：`一个人有问题，学到东西，最后变好了。` 这类写法只讲主题，不讲戏剧机制。
- 不是唯一范式：如果项目本来就是 chamber piece 或 mood film，写法会不同。

### 2. 剧集试播集 / 机制作业
- 适用：需要能反复出集的系列项目。
- 参考片段：`每周一，调解员打开屋顶诊所的候补名单，而整栋楼的电又开始跳闸。`
- 为什么更好：重复压力源是清楚的，观众能理解为什么这能周更。
- 为什么弱写法不行：`这部剧里有一群有趣的人，每集都会遇到新麻烦。` 这没有系统。
- 不是唯一范式：有些剧集更适合季节性累积，而不是周更回圈。

### 3. 短剧钩子型冲刺
- 适用：短时长、强钩子、快升级的场景。
- 参考片段：`她看完消息，笑了一下，把手机删了，还是打开了门。`
- 为什么更好：钩子马上到，反转是可见的。
- 为什么弱写法不行：`前面慢慢介绍人物，最后会有一个惊喜。` 对短剧来说太慢。
- 不是唯一范式：有些短剧靠气氛、重复或情绪压迫取胜。

### 4. 动画世界规则
- 适用：世界观本身要制造戏剧冲突的动画项目。
- 参考片段：`天一亮，村子里只要谁说过谎，影子上就会留下可见的污痕。`
- 为什么更好：规则本身就是戏剧装置。
- 为什么弱写法不行：`这是一个很有想象力、角色很多的世界。` 只是在说风格。
- 不是唯一范式：动画也可以很克制、很写实、很情绪化。

### 5. 悬疑 / 惊悚揭示链
- 适用：必须持续修正观众判断的故事。
- 参考片段：`他中午偷走的那把钥匙，打开的是整栋楼都装作不存在的房间。`
- 为什么更好：谜物、隐藏空间、后果都在。
- 为什么弱写法不行：`这个故事很紧张，最后会让你一直猜。` 只是在描述效果。
- 不是唯一范式：惊悚也可以是张力优先，不一定是谜题优先。

### 6. 爱情 / 关系确认
- 适用：关系变化要比“互相喜欢”更具体的爱情故事。
- 参考片段：`他在众人面前纠正她的玩笑，等房间空了，又用她真正想表达的方式把那句话重复了一遍。`
- 为什么更好：先有摩擦，再有理解。
- 为什么弱写法不行：`他们经历了很多可爱瞬间，然后相爱了。` 太泛。
- 不是唯一范式：爱情也可以是慢热、错位、反讽型。

### 7. 喜剧 setup / payoff
- 适用：笑点要落在明确的动作与反转上。
- 参考片段：`他把道歉备忘录打印出来，递错了老板，结果因为“诚实”被提前升职。`
- 为什么更好：setup 和 payoff 都是可见的。
- 为什么弱写法不行：`这一场大家都很搞笑。` 这不是喜剧机制。
- 不是唯一范式：喜剧也可以很冷、很钝、很人物向。

### 8. 严肃 / 作者向的留白
- 适用：空白本身是审美的一部分，但不能丢掉因果。
- 参考片段：`饭桌上没人说“战争”这个词，但每只盘子都被一个不再属于这里的人端走。`
- 为什么更好：留白存在，但压力仍然可读。
- 为什么弱写法不行：`这个故事很 subtle，观众应该看完后好好想想。` 这是空话。
- 不是唯一范式：留白不是缺戏；也不是每个项目都该追求。

### 9. 商业单讯息脚本
- 适用：广告、带货、单点转化型脚本。
- 参考片段：`一口下去，正在揉眼睛的通勤者第一次把整句话说完整了。`
- 为什么更好：利益点是可见动作，不是口号。
- 为什么弱写法不行：`我们的产品有很多优秀功能。` 这是卖点堆砌。
- 不是唯一范式：有些 campaign 要的是品牌感，不是立即转化。

### 10. 品牌片 / 品牌叙事
- 适用：品牌价值要通过故事出现，而不是靠标语出现。
- 参考片段：`她开车穿过被雨水压低的隧道，不是为了逃离风暴，而是为了带着工具赶去重建街区。`
- 为什么更好：品牌价值被动作演出来了。
- 为什么弱写法不行：`这个品牌代表坚韧、创新和关怀。` 这只是自我介绍。
- 不是唯一范式：品牌片可以很静、很像纪录片、也可以很夸张。

### 11. 短视频 / 竖屏开头
- 适用：首秒就要抓住人。
- 参考片段：`她把坏掉的搅拌机举到镜头前，按一下按钮，厨房突然安静了。`
- 为什么更好：先证明，再解释。
- 为什么弱写法不行：`我来讲一个能改善生活的产品。` 先把钩子浪费掉了。
- 不是唯一范式：短视频也可以靠悬念、人格或情绪取胜。

### 12. 游戏叙事 / 任务循环
- 适用：有重复循环和状态变化的游戏叙事。
- 参考片段：`每次你把遗物送回营地，营地都会变，但每次升级也都会关掉一条路。`
- 为什么更好：任务和状态之间有 tradeoff。
- 为什么弱写法不行：`玩家去做任务，并了解更多世界观。` 没有系统。
- 不是唯一范式：游戏也可以靠探索、社交或生存系统。

### 13. 分支互动 / 选择后果
- 适用：选择必须真的改变后续状态。
- 参考片段：`如果她撒谎，守卫会信她；如果她说真话，守卫会永远记住她的脸。`
- 为什么更好：后果是真后果，不是换台词。
- 为什么弱写法不行：`玩家会做出有趣的选择，看到不同场景。` 这没有后果。
- 不是唯一范式：分支可以浅、可以深，也可以偏状态型。

### 14. 改编 / 容器转换
- 适用：从原作、IP 或其他媒介转到新容器。
- 参考片段：`小说的秘密不是那封失踪的信，而是谁在每次死亡之后都重新写信。`
- 为什么更好：先找核心引擎，再谈容器转换。
- 为什么弱写法不行：`我们要忠于原作，尽量保持改编的准确性。` 这没有说明忠实度的标准到底是什么。
- 不是唯一范式：有些改编要贴近，有些改编要大改。

### 15. 重写诊断 / 剧本医生
- 适用：先诊断哪里坏了，再决定怎么修。
- 参考片段：`第一幕不是慢，而是主角没有做出一个会改变棋盘的决定。`
- 为什么更好：直接指出失败层。
- 为什么弱写法不行：`这稿子不够好，需要改进。` 没有指出问题所在。
- 不是唯一范式：重写可以优先修结构、人物、对白、节奏或类型满足度。

### 16. 对白打磨
- 适用：台词要区分声线和潜台词。
- 参考片段：`A: 你来晚了。 B: 我是按重要性来的。 A: 那你确实来晚了。`
- 为什么更好：有不对称，有压力，有潜台词。
- 为什么弱写法不行：`A: 我很生气。 B: 我理解你很生气。` 这只是把情绪说出来。
- 不是唯一范式：有些场景应该更省词，也有些应该更啰嗦。

### 17. 受众适配
- 适用：任务不是写故事，而是判断这故事给谁看、能不能卖得动。
- 参考片段：`这个版本适合想要悬疑加情绪释放的都市流媒体观众，但对需要更广泛 catharsis 的人群会偏窄。`
- 为什么更好：把受众、需求和风险都说清了。
- 为什么弱写法不行：`观众应该会喜欢，因为它很好。` 这是主观判断，不是 fit。
- 不是唯一范式：受众适配不是要求所有内容都变保守。

### 18. 开发简报
- 适用：需要先决定开发策略，而不是先写正文。
- 参考片段：`保留核心冲突，把揭示提前，把最深的反转留到第二段开发节点。`
- 为什么更好：把创作方向和开发逻辑连起来了。
- 为什么弱写法不行：`我们要把项目做得更强、更有市场性。` 这句话没有帮下一步决策。
- 不是唯一范式：简报是为了收敛选择，不是替代创作。

### 19. 编剧成长路径
- 适用：用户要的是训练路径，不是单次答案。
- 参考片段：`第 1 周写 logline，第 2 周写 scene function，第 3 周用同一场戏分别对两类 audience need state 重写。`
- 为什么更好：路径是分阶段、可检验的。
- 为什么弱写法不行：`多写、多看好剧本。` 这种建议无法执行。
- 不是唯一范式：学习路径要跟着能力缺口走，不是固定课程表。

### 20. 多路径方案
- 适用：当前问题还不该过早收敛。
- 参考片段：`A 路线押关系，B 路线押 mystery，C 路线押社会系统；三条路赢的方式不同。`
- 为什么更好：比较的是不同的赌注，不是同一想法的改写。
- 为什么弱写法不行：`这里给你三个版本，其实都差不多。` 这是假多样。
- 不是唯一范式：有些任务就应该快速收敛，不需要 path options。

### 21. 边界图与范围修正
- 适用：需要区分绝对不能做、可以大胆试、以及只是在某些条件下成立的说法。
- 参考片段：`这条规则不是全错，而是太宽了：在 festival drama 里还成立，在 brand work 里就不成立。`
- 为什么更好：保留了仍然成立的核心，也写出了失效边界。
- 为什么弱写法不行：`这条规则在一些时候对，在一些时候不对。` 太空泛。
- 不是唯一范式：边界图不是为了压死创意，而是为了避免把软约束写成硬边界。

## 这张图谱不是

- 不是模板库。
- 不是唯一真值表。
- 不是安全政策。
- 不是把创意锁死的标准答案。

它只是让一个请求的形状先变得清楚，这样后面的路由、协议、评分和返修才会更准。
